Descripcion del Proyecto

Chatbot conversacional que utiliza Retrieval Augmented Generation (RAG) para realizar busquedas semanticas inteligentes de vehiculos usados. Los usuarios describen lo que buscan en lenguaje natural y reciben recomendaciones personalizadas.

El Problema

Brecha Semantica
La busqueda tradicional no entiende intencion. "Auto para familia grande" no muestra camionetas.
Consultas Complejas
Necesidades multi-atributo dificiles de expresar con filtros.
Sin Contexto
Cada busqueda es independiente, sin memoria conversacional.

La Solucion: Arquitectura RAG

RAG combina busqueda semantica con IA conversacional.

1
Convertir a Texto Cada vehiculo se transforma en descripcion natural
2
Crear Embeddings OpenAI text-embedding-3-small (1,536 dimensiones)
3
Construir Vector Store FAISS indexa los 38,531 vehiculos
4
Consulta del Usuario Se embebe y busca por similitud semantica
5
Recuperar + Generar Top 5 resultados + GPT-4o-mini sintetiza respuesta

Ejemplo de Conversion

"Subaru Outback del ano 2010. Precio: $9,200 USD.
Motor: gasoline de 2.5 litros. Transmision: automatic.
Kilometraje: 190,000 km. Tipo de carroceria: universal.
Color: silver. Traccion: all. Estado: owned."

Ventajas de RAG

Super Rapido Busquedas en segundos
Economico 1 embedding + 1 llamada LLM
Busqueda Semantica Sin palabras exactas
Persistente Vector store en disco
Simple Menos trabajo que fine-tuning
Conversacional Memoria de contexto

Conclusiones

  1. RAG es ideal para recuperacion de datos
  2. Embeddings pre-entrenados capturan relaciones semanticas sin fine-tuning
  3. FAISS permite busqueda lista para produccion con infraestructura minima
  4. Memoria conversacional agrega valor para refinar busquedas
